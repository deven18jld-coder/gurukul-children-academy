# School Template - Automatic Image Management System

Welcome to the School Template! This project comes with a fully automated image management system. You don't need to touch any code to update the images on the website.

## How to Update Images

Simply drag and drop your images (`.jpg`, `.jpeg`, `.png`, `.webp`) into the corresponding folders inside `public/images/`. The website will automatically detect them and update the UI.

**Important:** Image filenames do not matter at all. Only the folder placement matters. You can upload `IMG_1234.jpg`, `DSC_5678.png`, or `my-image.webp`, and they will work perfectly based entirely on the folder they are placed in.

If a folder is empty, the website will gracefully fall back to its premium placeholder design.

### Folder Structure & Guidelines

| Folder Path | Description | Rules |
| :--- | :--- | :--- |
| `public/images/logo/` | School Logo (Header, Footer, SEO) | The **first** image found is used as the logo. |
| `public/images/hero/home/` | Home Page Hero Image | The **first** image found is displayed. |
| `public/images/hero/about/` | About Page Hero Image | The **first** image found is displayed. |
| `public/images/hero/admissions/` | Admissions Page Hero Image | The **first** image found is displayed. |
| `public/images/hero/facilities/` | Facilities Page Hero Image | The **first** image found is displayed. |
| `public/images/hero/gallery/` | Gallery Page Hero Image | The **first** image found is displayed. |
| `public/images/hero/contact/` | Contact Page Hero Image | The **first** image found is displayed. |
| `public/images/principal/` | Principal's Photo | The **first** image found is used in the Principal Message sections. |
| `public/images/teachers/` | Teachers Grid | **Every** image dropped here automatically creates a Teacher Card. The filename is used as the teacher's name. If the filename is generic (like `IMG_1234.jpg`), it will gracefully default to "Faculty Member". |
| `public/images/facilities/*` | Facilities Overview & Details | Drop images into subfolders like `classrooms`, `library`, `playground`, `transport`, `activity-room`, `safety`. The **first** image in each folder is used for that facility. |
| `public/images/gallery/*` | Gallery Grid | Drop multiple images into subfolders like `campus-life`, `events`, `sports`, `activities`, `celebrations`. **All** images are loaded into the Gallery grid automatically. |

## Development Commands

| Command                   | Action                                           |
| :------------------------ | :----------------------------------------------- |
| `npm install`             | Installs dependencies                            |
| `npm run dev`             | Starts local dev server at `localhost:4321`      |
| `npm run build`           | Build your production site to `./dist/`          |

*Note: The image manifest (`src/generated/image-manifest.json`) is automatically generated by `scripts/generate-image-manifest.js` during the dev and build processes.*
